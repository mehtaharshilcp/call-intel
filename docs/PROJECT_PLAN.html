Project Plan

Full detail and Mermaid diagram: docs/PROJECT_PLAN.md. Prompts: docs/PROMPT_LOG.md.

1. Requirements

Pipeline: Upload → transcribe → speaker / talk-time → LLM analysis → storage → dashboards.

Main dashboard: Total calls, sentiment split, avg score (0–10), avg duration, top keywords, total action items.

Per-call: Summary, sentiment, synced transcript + audio, talk-time %, overall score, five 1–10 dimensions, questionnaire coverage, keywords, action items, positive/negative observations.

2. Logical data model

  • Call — id, filename, duration, status, errors, timestamps
  • TranscriptSegment — times, speaker (agent/customer/unknown), text
  • CallAnalysis — sentiment, scores, summary, questionnaire map, keywords, actions, observations, evidence optional
  • Questionnaire — seeded topics with hints for LLM coverage

3. Pipeline steps

  1. Upload & validate
  2. STT → segments (+ optional LLM speaker labels)
  3. Talk-time % from segments
  4. Analysis LLM → JSON → validate / retry
  5. Aggregate for main dashboard

4. Runtime endpoints (IndexedDB-only app)

Note: This project uses IndexedDB only. No external database is used.

MethodPathRole

5. Frontend pages

  • Calls + upload
  • Main dashboard
  • Call detail (audio sync, all analysis sections)

6. Phases

Scaffold → DB/seed → upload/list → STT → LLM analysis → dashboard API → UI → polish (errors, reprocess, demo data).

7. Success criteria

  • Upload → ready
  • Dashboard aggregates with multiple calls
  • Detail: sync transcript, questionnaire, scores, keywords, actions, observations
